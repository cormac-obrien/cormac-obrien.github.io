Introduction

Lexical Analysis

Lines

MICRON programs are divided into logical lines, each separated by a NEWLINE token. A logical line can span multiple physical lines, which are separated by an ASCII line feed.

Comments

MICRON uses C-style comments. Comments begin at /* and end at */.

/* This is a comment. */

/* This is also *
 * a comment.   */

There are no single-line comments in MICRON.

Encoding

UTF-8 is the sole encoding supported by MICRON.

Line Joining

Lines in MICRON may be explicitly joined by placing a backslash \ at the end of a line.

if x == 5 \
  and y == 6:
    /* do something */

MICRON implicitly joins lines inside delimiters. This means that array and struct definitions can span multiple lines.

Indentation

MICRON uses spaces for indentation. Hard tabs should not appear in source code. One level of indentation is four spaces. Indentation is the method for determining blocking/scoping. Anonymous scoping is permitted simply by indenting four spaces and dedenting when the block should be finished.

Keywords, Identifiers and Literals

Keywords

The following words are reserved as keywords and may not be used for any other purpose.

and do elif else fn for if import not or struct while
